这份方案的目标是：**通过一套极简的文本协议（DSL），驱动一套工业级的渲染回放引擎（AlphaTab），解决从“文字”到“专业视听乐谱”的最后一公里。**

## 🎼 Lyrichord (歌弦) 开源项目全栈设计方案

### 1. 技术底座 (The "Frankenstein" Stack)

我们不造任何轮子，只做最强的集成商：

* **渲染与回放引擎**: **AlphaTab** (GPL-3.0)。它是目前网页端唯一能完美解析、渲染并同步播放六线谱的引擎，支持 AlphaTex（类 LaTeX 协议）。
* **音频合成**: **AlphaSynth** (内置)。它基于 Web Audio API，支持加载 **SoundFont (SF2)**，能真实模拟吉他的延音、切音和力度。
* **文本编辑器**: **Monaco Editor**。支持自定义语法高亮和 Lint 检查。
* **跨平台容器**: **Vite + React/Vue 3**。

---

### 2. Lyrichord DSL 深度规范 (TMD - TabMarkdown)

我们需要定义一种非二进制、Git 友好的纯文本格式。

#### **A. 头部声明 (Header)**

```yaml
---
title: 你瞒我瞒 (Lyrichord Test Case)
tempo: 72
time_signature: 4/4
capo: 2
# 预定义节奏型映射 (Rhythm Matrix)
@R1: "p - 3 - (1 2) - 3"          # 分解版：四分音符步长
@R1+: "p 3 2 1 2 3 (1 2) 3"      # 密集版：八分音符步长
@R2A: "D . . . | D . D U | X . . . | D . D U" # 呼吸扫弦 (X为休止)
@R2B: "D . . . | D . D U | X U D U | D . D U" # 爆发扫弦
---

```

#### **B. 指法字典 (The Chord DB)**

系统内置标准库，但允许就地覆盖特殊指法：

```yaml
# 允许用户临时定义极其复杂的指法
define [D/#F]: { frets: "2 0 0 2 3 2", fingers: "T 0 0 1 3 2" }
define [B7]: { frets: "x 2 1 2 0 2", fingers: "0 2 1 3 0 4" }

```

---

### 3. 核心算法：转译器 (The Transpiler Logic)

这是整个项目的核心逻辑：**如何把 `[C] @R1` 变成乐谱上的音符坐标。**

#### **第一步：时间轴切分 (Time Slicing)**

* AlphaTab 的底层逻辑是 Ticks (1/4 拍 = 480 Ticks)。
* `@R1` 有 4 个元素 -> 每个元素时值 = 480 Ticks。
* `@R1+` 有 8 个元素 -> 每个元素时值 = 240 Ticks。

#### **第二步：物理坐标映射 (Physical Mapping)**

针对 `[C]` 和弦指法 `[x 3 2 0 1 0]`：

* **指令 `p` (Root)**：算法自动寻找该和弦非 `x` 的最底弦（这里是 5 弦 3 品）。
* **指令 `(1 2)` (Simultaneous)**：识别为同时触发 1 弦和 2 弦（0 品和 1 品）。
* **指令 `X` (Articulation)**：生成 `r` (Rest) 指令，或在音频层发送一个 `NoteOff` 信号。

#### **第三步：AlphaTex 生成 (Standard Output)**

转译器将 DSL 展开为 AlphaTex 字符串：
`\chord "C" "x 3 2 0 1 0" 3.5 0.3 (1.2 0.1) 0.3 |`

---

### 4. 复杂技巧的工程实现 (Articulations)

为了支持“击、勾、扫、切”，转译层必须支持装饰符：

| 技巧 | DSL 表达 | 逻辑处理 | AlphaTex 输出 |
| --- | --- | --- | --- |
| **击弦 (Hammer-on)** | `1 H 2` | 在 1 弦 1 品与 2 品间加连奏标记 | `1.1 L 2.1` |
| **扫弦 (Strum)** | `D / U` | 添加 `BrushDown` / `BrushUp` 标记 | `(和弦坐标) {bd}` |
| **死音 (Dead Note)** | `x` | 将品位替换为 `x` | `x.5` |
| **切音 (Slap)** | `@S` | 触发拍弦采样点 | `(和弦坐标) {slap}` |

### 5. 第一首测试曲目实现：下行《你瞒我瞒》

我们要实现你说的“A B 段进入两次，第二次才是扫弦”的逻辑：

#### **DSL 输入层：**

```text
[A1] @R1
(C)约会像是为(D)分享到饱肚滋(G)味
(C)有任何难(D)题却不提(Em)起

[A2] @R1
(C)我是谁情(B7)人你始终也(Em)是你

# 第二轮切换逻辑
[A2_v2] @R2A
(C)这就是谈(B7)情客气得吓着我
(D/#F)除了近来(Em)繁忙 (C)我所知(D)有几(G)多

```

#### **渲染层逻辑：**

1. **动态节奏切换**：当解析器扫描到 `[A2_v2]` 块时，它会查表获取 `@R2A`。
2. **呼吸感实现**：`@R2A` 里的 `X` 占位符会让渲染出的乐谱在第三拍出现一个休止符，同时 AlphaSynth 停止当前音轨的延音，模拟出那种客气、断断续续的“呼吸感”。

---

### 6. 系统详细模块清单

1. **`TmdParser` (核心解析)**：基于正则表达式或 PEG.js，将 TMD 文本转为中间抽象语法树 (AST)。
2. **`LayoutEngine` (排版引擎)**：计算歌词与和弦的对齐位置。如果一行和弦太多，自动折行。
3. **`AudioBuffer` (采样管理)**：加载 `acoustic_guitar_steel.sf2`。
4. **`LivePreview` (实时预览)**：使用 Web Worker 进行后台转译，主线程渲染 Canvas，保证输入不卡顿。
5. **`ExportModule`**：利用 AlphaTab 的截图功能导出为 SVG/PDF。

### 7. 为什么要这么做？(技术优势)

* **极速录入**：你再也不用在 Guitar Pro 里一个音符一个音符地拉。你只要把歌词粘进来，在括号里填和弦，剩下的交给算法。
* **版本控制**：乐谱变成了纯文本。你可以用 Git 记录你对《你瞒我瞒》的每一次节奏修改。
* **动态交互**：用户可以直接点击谱面上的任何一个小节，AlphaTab 会精确定位到对应的歌词行进行播放。

### 8. 项目 Roadmap (MVP 阶段)

1. **Phase 1**: 跑通 AlphaTab + Monaco Editor 的基础联动，支持渲染简单的 AlphaTex。
2. **Phase 2**: 实现 `Pattern -> Note` 转换算法，支持基本的分解和扫弦模板。
3. **Phase 3**: 录入《你瞒我瞒》完整 TMD 脚本，优化 D/#F 和 B7 等和弦的指法库。
4. **Phase 4**: 优化音频回放，加入 SF2 采样，实现你要求的扫弦听感。
