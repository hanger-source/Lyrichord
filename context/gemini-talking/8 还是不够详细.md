明白，作为开发者，你需要的是一份**“可以直接写进 README 和技术设计文档”**级别的详细方案。我们要解决的是如何从一套**类似 Markdown 的 DSL**，精准地映射到 **AlphaTab 的数据结构**中，而不依赖任何外部 GP 文件。

以下是 **Lyrichord (歌弦)** 的全栈架构与核心实现算法的详尽细节。

---

### 一、 系统核心架构图 (Data Pipeline)

1. **输入层 (DSL)**: 用户在 Monaco Editor 输入 `.tmd` 文本。
2. **解析层 (Parser)**: 将文本拆解为 `Header`（节奏定义）和 `Body`（歌词和弦）。
3. **转译层 (Generator)**: 将 DSL 转换为 **AlphaTex** 字符串（AlphaTab 的官方标注语言）。
4. **渲染层 (Renderer)**: AlphaTab 接收 AlphaTex，调用 Canvas/SVG 绘图。
5. **音频层 (Audio)**: AlphaSynth 加载 SF2 采样，根据 AlphaTex 里的音符时值进行 MIDI 触发。

---

### 二、 核心 DSL 语法设计 (TabMarkdown)

为了极致的输入效率，我们将语法设计为：

```yaml
---
# 1. 节奏型模板库 (定义一次，全局调用)
@R1: pluck(p-3-12-3, speed=1.0)        # 分解：根音-3弦-1&2弦-3弦
@R1+: pluck(p-3-2-1-2-3-12-3)          # 密集分解
@R2A: strum(D---D-DU, pause_at=3)      # 呼吸扫弦：第3拍留白
@R2B: strum(D---D-DU, syncopated=true) # 爆发扫弦
---

# 2. 演奏区
{section: A1, rhythm: @R1}
(C)约会像是为(D)分享到饱肚滋(G)味
(C)有任何难(D)题却不提(Em)起

{section: B1, rhythm: @R2A}
(C)无言的亲亲(D)亲 侵袭我(G)心

```

### 三、 算法实现：如何“缝合”到 AlphaTab？

这是项目最难的部分：**如何把 `@R1` 变成乐谱上的音符？**

#### 1. 和弦指法映射表 (Chord Database)

你需要一个 JSON 字典来存储和弦：

```json
{
  "G": { "name": "G", "frets": [3, 2, 0, 0, 0, 3], "fingers": [3, 2, 0, 0, 0, 4] },
  "D/#F": { "name": "D/#F", "frets": [2, 0, 0, 2, 3, 2], "bass": 6 }
}

```

#### 2. 节奏型展开算法 (The Expansion Logic)

当解析到 `(C) @R1` 时，转换器执行以下逻辑：

* **Step A (确定时值)**: 假设 4/4 拍，一小节 8 个八分音符位。`@R1` 是 4 个动作，则每个动作占 **1/4 拍 (四分音符)** 或 **1/8 拍 (八分音符)**。
* **Step B (物理坐标映射)**:
* `p` (根音): 自动查找和弦的最低有效音。如 C 和弦在 5 弦 3 品。
* `3`: 查找 C 和弦在 3 弦的品位（0 品）。
* `12`: 同时触发 1 弦和 2 弦的品位。


* **Step C (输出 AlphaTex)**:
转换后的 AlphaTex 代码段：`3.5 0.3 (1.2 0.1) 0.3`（格式为：品位.弦号）。

#### 3. 扫弦模拟算法 (Strumming Simulation)

扫弦不是简单的拨片划过，AlphaTab 需要 `brush` 指令：

* **D (Down)**: `(3.5 2.4 0.2 1.2 0.1) {bd}` (bd 代表 Brush Down)。
* **U (Up)**: `(3.5 2.4 0.2 1.2 0.1) {bu}` (bu 代表 Brush Up)。
* **X (Pause)**: 插入一个 `r` (Rest) 指令，时值为当前步长。

### 四、 详细模块设计文档

#### 模块 A: `TmdParser.js` (DSL 解析器)

* **职责**: 使用正则表达式捕获 `{}` 里的元数据和 `()` 里的和弦。
* **输出**: 一个内部 JSON 对象，描述每一行歌词对应的节奏型和和弦序列。

#### 模块 B: `AlphaTexGenerator.js` (逻辑转译器)

* **职责**: 核心“缝合”逻辑。
* **关键函数**: `generateMeasures(internalJson)`。
* **细节**: 它需要处理“小节平衡”。如果一行歌词有 4 个和弦，它必须自动将这一行切分为 2 个小节或 4 个小节，以保证 AlphaTab 渲染不重叠。

#### 模块 C: `AudioEngine.js` (声音驱动)

* **职责**: 管理 AlphaSynth。
* **配置**:
```javascript
const player = new alphaTab.synth.AlphaSynth(api);
player.loadSoundFont('/path/to/guitar_samples.sf2'); // 关键：高品质采样

```

### 五、 针对《你瞒我瞒》的特别渲染优化

为了让这首歌在你的平台上“一跑就惊艳”，需要特殊处理：

1. **低音下行高亮 (Bass Line Focus)**:
在解析 `G -> D/#F -> Em` 时，在 AlphaTex 层面给这些根音添加 `\f` (Forte) 强音标记，让合成器把这几个低音弹得更重。
2. **情感色阶 (Dynamic Scaling)**:
为不同节奏型设置默认音量权重：
* `@R1`: `volume: 0.5`
* `@R2B`: `volume: 1.0`
渲染时，在 AlphaTex 开头加入 `\v 60` 或 `\v 110` 指令。

### 七、 为什么叫 "Lyrichord"？

因为它把 **Lyrics** (感性的歌词) 和 **Chord** (理性的和弦) 结合在了一起。你的项目将成为程序员学吉他的神器——**因为调音色和改谱子现在就像改代码一样简单。**
