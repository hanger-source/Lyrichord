
这是一个非常典型的**“工程师思维”**：不重复造轮子，通过高效集成（缝合）业界成熟的开源库，快速交付一个满足特定场景（文本驱动、实时渲染、自动回放）的最小可行性产品（MVP）。

以下是为你量身定制的 **“Lyrichord” (歌弦)** 项目开发方案。

---

## 1. 项目背景与目标

* **痛点**：传统的图片谱没有声音，Guitar Pro 软件太重且不便于分享，ABC Notation 对吉他指法不友好。
* **目标**：构建一个基于 Web 的轻量级吉他谱平台。用户只需输入类似 Markdown 的**文本（歌词 + 和弦 + 节奏型代号）**，系统自动渲染出专业六线谱，并能根据节奏型算法自动合成吉他伴奏音频。
* **首发 Demo**：陈柏宇《你瞒我瞒》高阶情感阶梯版。

---

## 2. 硬核“缝合”技术栈

我们要做的不是重新发明乐谱，而是做一层 **DSL (领域特定语言) 到 专业引擎** 的映射。

| 模块 | 推荐开源库 | 理由 |
| --- | --- | --- |
| **渲染 & 解析引擎** | **[AlphaTab](https://www.alphatab.net/)** | **核心支柱**。支持浏览器端解析 GP 文件、渲染六线谱、同步播放。 |
| **音频合成** | **[AlphaSynth](https://www.google.com/search?q=https://www.alphatab.net/docs/alphasynth/intro)** | AlphaTab 自带的合成器，支持加载 **SoundFont (音色库)**，声音真实。 |
| **DSL 解析器** | **[Peg.js](https://www.google.com/search?q=https://pegjs.org/)** 或 正则 | 将你自定义的文本语法（如 `[C]约会`）解析成 AlphaTab 的 JSON 数据。 |
| **前端框架** | **Vue 3 / React** | 用于构建左边编辑、右边预览的 IDE 界面。 |
| **编辑器** | **Monaco Editor** | VS Code 的内核，提供极佳的代码输入体验和语法高亮。 |

---

## 3. 核心数据结构：TabMarkdown (TMD)

我们要定义一套极简的文本协议。

### **语法示例 (`you_man_wo_man.tmd`)**

```yaml
# 声明节奏型模板 (Rhythm Templates)
@R1: { pattern: "p i (m a) i", speed: 1.0 }  # 节奏型1: T-3-(12)-3
@R1Plus: { pattern: "p i m a m i (m a) i" } # 节奏型1+: T-3-2-1-2-3-(12)-3
@R2A: { pattern: "D - - - | D - D U | X - - - | D - D U" } # 扫弦呼吸版

# 歌曲内容
[Verse 1] @R1
[C]约会像是为[D]分享到饱肚滋[G]味
[C]有任何难[D]题却不提[Em]起

[Chorus 1] @R1Plus
[C]无言的亲亲[D]亲 侵袭我[G]心

```

---

## 4. 系统架构方案

### **Step 1: 文本预处理器 (The Glue)**

你需要写一个转换逻辑（Parser），将上面的文本转换成 AlphaTab 能读懂的 **AlphaTex**（AlphaTab 的专用轻量级标注语言）。

* **逻辑**：当检测到 `[C]` 时，在谱面上放置一个 C 和弦。
* **难点**：将节奏型代号（如 `@R1`）展开为具体的音符时值。例如 `@R1` 展开为四个八分音符：根音、3弦、12弦同时、3弦。

### **Step 2: 渲染引擎集成 (AlphaTab)**

```javascript
// 初始化 AlphaTab
const api = new alphaTab.AlphaTabApi(element, {
    core: {
        engine: 'html5',
        fontDirectory: '/font/'
    },
    player: {
        enablePlayer: true,
        soundFont: '/soundfonts/acoustic_guitar.sf2' // 加载高音质吉他采样
    }
});

```

### **Step 3: 自动播放逻辑**

AlphaTab 的播放器是基于谱面音符的。

* 通过你的 DSL 解析，你已经在 JSON 数据里填好了音符。
* 点击 Play，AlphaSynth 会根据你设定的 `bpm` 和节奏型，自动触发对应的 Sample（采样点）。

## 5. 首个测试曲目：下行《你瞒我瞒》逻辑实现

为了实现你要求的“不突兀”和“匹配调”，我们需要在解析层做特殊映射：

* **根音映射表**：
* `G` -> 6 弦
* `C` -> 5 弦
* `D/#F` -> 6 弦 2 品


* **节奏型动态切换**：
* 在解析到 `[Chorus]` 标签时，指令 API 切换播放器的 `pattern` 权重。
* **扫弦模拟**：在 AlphaTab 中，通过 `brushUp` 和 `brushDown` 属性来模拟型 2-A 的呼吸感。

## 6. 开发 Roadmap (MVP 阶段)

1. **Phase 1 (底层)**: 跑通 AlphaTab Demo，加载一个本地的 `.gp5` 文件确保渲染和声音正常。
2. **Phase 2 (DSL)**: 实现一个基础正则解析器，能把 `[C] [D] [G]` 文本转换成对应的 AlphaTab 和弦图。
3. **Phase 3 (节奏引擎)**: 实现“节奏型代号”到“音符序列”的映射。这是最关键的一步，让用户输入 `@R1` 就能看到你设计的那 4 个动作。
4. **Phase 4 (UI)**: 加上双栏预览界面。

### **项目起名建议：**

* **Lyrichord (歌弦)**：优雅、直观。
* **TabStack (谱栈)**：技术感强，适合开源。
* **GitTab**：强调像 Git 一样管理文本曲谱。
