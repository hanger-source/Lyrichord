## 🛠 Lyrichord (歌弦) 工业级全栈技术方案

### 第一章：DSL 深度语法规范 (Lyrichord-TMD)

这是用户交互的最前端。我们要支持的不仅是简单的字符，而是带有**音乐属性**的结构化文本。

#### 1.1 技巧标注集 (Articulation Syntax)

为了支持击弦、勾弦、死音等，我们定义一套微型符号：

* **H/P (Hammer/Pull)**: `1h2` (1品击向2品), `3p0` (3品勾向0品)。
* **S (Slide)**: `2s4` (2品滑向4品)。
* **X (Dead Note)**: `x` (右手切音或左手闷音)。
* **B (Bend)**: `2b4` (2品推弦至4品音高)。
* **M (Palm Mute)**: `(p)m` (低音弦手掌闷音)。

#### 1.2 节奏模板动态逻辑

节奏型不再是死板的字符串，而是一个**时值数组**。

* **@R1 (4/4拍, 8分音符步长)**: 分配 8 个 Slot。
* **语法**: `Slot1 | Slot2 | Slot3 | ...`
* **解析规则**: 遇到 `-` 表示延续（延音），遇到 `r` 表示休止。

### 第二章：核心转译器实现 (Transpiler Engine)

这是最复杂的逻辑层，负责将 DSL 转换为 AlphaTab 的 **AlphaTex** 指令。

#### 2.1 时值计算模型 (Tick Math)

AlphaTab 默认一小节为 **1920 Ticks** (4/4 拍)。

* **算法**: `TickPerSlot = 1920 / Template.length`。
* **实现**: 如果用户在 `@R1` 定义了 8 个动作，每个动作分配 240 Ticks。解析器必须精确对齐这些 Tick，否则回放会音画不同步。

#### 2.2 状态机转换逻辑

```javascript
// 伪代码：解析 [C] @R1
function translate(chord, pattern) {
    const fingering = ChordLib.get(chord); // 获取 [x, 3, 2, 0, 1, 0]
    return pattern.slots.map(slot => {
        if (slot === 'p') return getRootNote(fingering); // 映射到 5弦3品
        if (slot === '12') return getSimultaneous([1, 2], fingering); // 映射到 1,2弦
        if (slot === 'h') return injectHammerOn(); // 插入 AlphaTex 的 'L' 指令
    }).join(' ');
}

```

### 第三章：和弦与指法系统 (Chord & Voicing)

处理特殊和弦（Slash Chords）和自定义指法。

#### 3.1 复合和弦处理 (Slash Chords)

针对 `D/#F`：解析器执行 `split('/')`。

* **主部 (D)**: 决定高音弦的指法。
* **根部 (#F)**: 覆盖原本的根音位置，强制在 6 弦 2 品寻找 `p` 指令。

#### 3.2 自定义指法逃逸符

如果用户按了一个奇怪的指法：`[MyChord]{x 5 7 7 6 0}`。

* 解析器跳过内置库，直接提取大括号内的数组作为该小节的渲染坐标。

### 第四章：AlphaTab 渲染深度缝合 (Rendering Engine)

AlphaTab 是一个复杂的 Canvas 引擎，我们需要通过 **AlphaTex** 协议与其通讯。

#### 4.1 AlphaTex 指令映射表 (Mapping Table)

| 物理技巧 | AlphaTex 语法示例 | 说明 |
| --- | --- | --- |
| **扫弦下** | `(3.5 2.4 0.3) {bd}` | `bd` 触发向下的箭头渲染 |
| **扫弦上** | `(3.5 2.4 0.3) {bu}` | `bu` 触发向上的箭头渲染 |
| **击/勾弦** | `3.2 L 5.2` | `L` 自动计算 H 或 P 弧线 |
| **三连音** | `(4.3 2.3 0.3) {tu 3}` | 强制将三个音符合并为三连音时值 |

#### 4.2 实时 diff 渲染优化

为了不让输入时闪烁，前端需维护一个 **Virtual Score State**，只在 DSL 发生有效变化（如和弦改变、节奏型切换）时，才调用 `api.tex()`。

### 第五章：音频合成与采样 (Audio Engine)

这是让《你瞒我瞒》好听的关键：**基于 SF2 的 AlphaSynth**。

#### 5.1 采样层级 (Sample Layers)

* **SoundFont 加载**: 加载一个带 **Velocity Layers (力度层)** 的吉他 SF2。
* **力度映射**:
* 主歌 (@R1): 发送 MIDI Velocity 50-60。
* 高潮 (@R2B): 发送 MIDI Velocity 100-110。
* 这将触发 SF2 中不同的采样点（扫弦更响、拨弦更柔）。



#### 5.2 效果器链 (Effect Chain)

AlphaSynth 支持基本的 **Reverb (混响)** 和 **Master Volume**。

* 在第一段（A1 段）开启微量混响，营造“内心独白”的空灵感。
