这是一份为您定制的、工业级深度的 **Lyrichord (歌弦)** 开源项目技术设计白皮书。这份方案涵盖了从 DSL 语法解析到底层音符映射的所有细节，确保您在开发时能够直接作为逻辑参考。

---

## 1. 系统架构总览 (System Architecture)

该平台采用“**三层抽象**”架构，实现从感性文本到专业音频的转换。

* **Layer 1: Input Layer (Lyrichord DSL)**: 用户输入的文本，易于阅读和版本管理。
* **Layer 2: Logic Layer (The Transpiler)**: 核心算法，将 DSL 翻译为 AlphaTex 协议。
* **Layer 3: Engine Layer (AlphaTab + AlphaSynth)**: 负责六线谱的高精度渲染和基于采样（SF2）的声音合成。

---

## 2. Lyrichord DSL 详尽定义

为了支持您提到的“特殊和弦、复杂节拍、击弦扫弦”，DSL 必须结构化。

### 2.1 头部声明 (Metadata & Templates)

使用 YAML 风格声明全局参数。

```yaml
---
title: 你瞒我瞒
author: 陈柏宇
bpm: 72
time_signature: 4/4  # 支持 3/4, 7/8 等任意节拍
capo: 2

# 技巧模板定义
@R1: pluck(p-3-12-3)                 # 基础分解
@R1Plus: pluck(p-3-2-1-2-3-12-3)     # 密集分解
@R2A: strum(D-X-D-DU)                # 呼吸扫弦 (X 代表呼吸留白)
@R_Hammer: pluck(p-3-2-1H2-2-3)      # 击弦示例 (1H2 代表在1弦从1品击向2品)
---

```

### 2.2 和弦库定义 (Chord Dictionary)

系统内置标准库，但也允许在文中自定义特殊指法。

* 语法：`(和弦名){6弦品位 5弦 4弦 3弦 2弦 1弦}`
* 示例：`(D/#F){2 0 0 2 3 2}`

## 3. 核心算法：从 DSL 到 AlphaTex 的映射

这是“不造轮子”的关键。你需要编写一个转译器，将用户的简写展开成 AlphaTab 的官方协议。

### 3.1 节拍与时值计算

在 4/4 拍中，我们将一小节划分为 **480 Ticks**（标准 MIDI 长度）。

* 若 `@R1` 有 4 个动作，则每个动作分配 120 Ticks（即一个四分音符）。
* 若 `@R1Plus` 有 8 个动作，则每个动作分配 60 Ticks（即一个八分音符）。

### 3.2 技巧映射表 (Technique Mapping)

你需要将 DSL 里的符号精准映射到 AlphaTex 的指令集：

| 技巧 | DSL 符号 | AlphaTex 指令 | 说明 |
| --- | --- | --- | --- |
| **拨弦** | `p, i, m, a` | `弦.品位` | 查找对应和弦指法坐标 |
| **扫弦** | `D, U` | `(坐标) {bd}`, `(坐标) {bu}` | `bd` 是 Brush Down, `bu` 是 Brush Up |
| **击弦** | `H` | `音符1 L 音符2` | `L` 在 AlphaTab 中表示 Legato 连奏 |
| **勾弦** | `P` | `音符1 L 音符2` | 同上，渲染引擎自动根据音高判断 H/P |
| **切音/死音** | `X` | `x.弦` | 渲染为 X 符号，发出短促打击声 |
| **闷音** | `PM` | `音符 {pm}` | Palm Mute，发出低沉闷音 |

## 4. 针对《你瞒我瞒》的逻辑转译实例

以第一句为例：`(C) @R1`

1. **解析和弦**：`C` -> 指法 `[x, 3, 2, 0, 1, 0]`。
2. **解析节奏**：`@R1` 定义为 `p-3-12-3`。
3. **计算坐标**：
* `p` (根音) -> 5 弦 3 品 -> `3.5`
* `3` (3弦) -> 3 弦 0 品 -> `0.3`
* `12` (1/2弦同时) -> `(1.2 0.1)`
* `3` (3弦) -> `0.3`


4. **生成 AlphaTex**：
`3.5 0.3 (1.2 0.1) 0.3`

如果是副歌的**呼吸扫弦** `(C) @R2A` (D-X-D-DU)：

1. **D**: `(3.5 2.4 0.3 1.2 0.1) {bd}`
2. **X**: `r` (Rest，休止符)
3. **D**: `(3.5 2.4 0.3 1.2 0.1) {bd}`
4. **DU**: `(3.5 2.4 0.3 1.2 0.1) {bd} (3.5 2.4 0.3 1.2 0.1) {bu}`

## 5. 详细开发模块拆解

### 模块 A: `TmdScanner` (语法扫描器)

* **输入**：Raw Text。
* **输出**：Tokens（识别出指令、和弦、歌词文字）。

### 模块 B: `MusicModel` (音乐模型)

* **功能**：这是你的“大脑”。它负责处理**节拍对齐**。例如：如果一行里有 2 个和弦，它必须确保这两个和弦平分这小节的时值。
* **逻辑**：如果用户在一行里写了太多和弦，模型自动根据节拍进行分小节（Bar Splitting）。

### 模块 C: `AlphaTabBridge` (渲染桥接器)

* **功能**：与 AlphaTab API 通讯。
* **代码实现**：
```javascript
import alphaTab from '@coderline/alphatab';

// 监听 DSL 改变
editor.on('change', (content) => {
    const texData = transpiler.generate(content); // 核心转译逻辑
    api.tex(texData); // 触发重新渲染
});

```



### 模块 D: `Sampler` (声音采样器)

* **职责**：加载音色库。
* **关键配置**：你需要一个包含吉他各种技巧采样的 **SoundFont (SF2)**。
* *提示*：AlphaSynth 会自动识别 AlphaTex 里的技巧（如 `{pm}`），并从 SF2 文件中寻找对应的样本。

## 6. 为什么这个方案能支持“复杂性”？

1. **多声部支持**：AlphaTab 支持多声部。如果你想支持更复杂的指弹（Bass 线和高音旋律线分离），你只需在生成 AlphaTex 时开启多声部语法。
2. **任意指法**：由于我们是基于“弦.品”的坐标系统，世界上任何奇怪的和弦（甚至是你乱按的）都能被渲染。
3. **动态节奏**：如果你需要“渐慢”或“切分音”，只需在时值分配算法中给特定的音符多分配一些 Ticks 即可。

1. **Setup**: 安装 `npm install @coderline/alphatab`。

4. **Audio**: 找到 `Acoustic_Guitar.sf2`，确保 `api.play()` 能出声。
5. **Test Case**: 录入《你瞒我瞒》前四句，验证拨弦到扫弦的切换是否顺滑。
